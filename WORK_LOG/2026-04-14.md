### 4. 敏感词门禁 Middleware

**文件**: `src/domains/security/sensitive-word-guard.ts`

**设计**：
- 复用 `ContentFilter` 做词检测逻辑
- 实现 `AgentMiddleware` 接口，通过 `agent.use()` 注册
- 非文本消息（GroupUpdated、ReadReceipt 等）直接透传，不干预

**中间件处理逻辑**：

```
incoming message
      │
      ▼
typeof content === "string"?
  NO  → next()，退出（透传系统消息）
  YES ↓
ContentFilter.check(content)
  BLOCKED → log BLOCKED + ctx.conversation.send(warningMessage) → return（拦截）
  PASSED  → next() → log PASSED + ctx.conversation.send(greetingMessage)
```

**关键技术细节**：
- 回复使用 `ctx.conversation.send(encodeText(...))` 而非 `ctx.sendTextReply()`
- `sendTextReply()` 发送 `Reply` content type（引用原消息的结构体），接收方的 `typeof ctx.message.content` 为 object 而非 string，导致显示为 `[non-text]`
- `ctx.conversation.send(encodeText(...))` 发送普通文本消息，接收方可直接读取 string content

**当前敏感词**（占位）：`['sb', '傻逼']`  
TODO：后端 API 就绪后替换为动态拉取 + 缓存机制。

## 关键技术决策记录

### XMTP db 加密问题

- XMTP 本地 SQLite 使用 SQLCipher 可选加密
- 加密与否的标志：目录下是否存在 `.sqlcipher_salt` 文件
- **混用加密/非加密打开同一个 db 文件会报** `unsupported file format`
- 不同钱包独立 dbDir，各自维护加密状态：
  - alice：无加密（`data/alice/`）
  - bob：有加密，密钥来自 `config/test-wallets.json` 的 `dbEncryptionKey` 字段

### XMTP Installation 限制

每次删除 `.db3` 重新 `Agent.create()` 会在 XMTP 网络注册新的 installation。  
**每个 inboxId 限制 5 个 installation**，超出后最旧的会被吊销。  
- alice 当前已有 4 个 installations（[WARNING] 已出现）
- **不要频繁删除 db 文件**，应持久化保存

### IdentifierKind const enum 双包冲突

`@xmtp/node-sdk` 和 `@xmtp/node-bindings` 各自定义 `IdentifierKind` const enum，  
两个包实例在运行时是不同对象，类型不兼容。  
**修复方法**：直接使用数字字面量 `0`，加类型断言 `as Identifier`：
```typescript
return { identifier: address, identifierKind: 0 } as Identifier;
```
